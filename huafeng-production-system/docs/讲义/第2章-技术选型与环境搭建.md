## 2.1 技术选型原则

### 选型考虑因素

| 因素 | 说明 |
| 项目|-----|------|
需求 | 是否满足功能要求 |
| 学习成本 | 团队是否熟悉 |
| 社区生态 | 文档、教程、插件是否丰富 |
| 性能 | 能否满足性能要求 |
| 维护性 | 代码是否易于维护 |
| 成本 | 是否有商业授权问题 |

### 本项目技术栈

| 层级 | 技术选型 | 选型理由 |
|-----|---------|---------|
| **前端框架** | Vue3 | 组合式 API、响应式系统强大 |
| **构建工具** | Vite | 启动快、热更新体验好 |
| **状态管理** | Pinia | 轻量级、TypeScript 支持好 |
| **路由** | Vue Router | Vue 官方路由 |
| **HTTP 客户端** | Axios | 功能强大、兼容性好 |
| **后端框架** | Express | 简单易用、生态丰富 |
| **数据库** | SQL.js | 无需安装、浏览器可直接运行 |
| **Excel 解析** | xlsx | 功能最全的 Excel 库 |
| **图表** | ECharts | 功能强大、文档完善 |

### 为什么选择这些技术？

#### Vue3 + Composition API

```javascript
// Vue2 Options API
export default {
  data() {
    return { count: 0 }
  },
  methods: {
    increment() {
      this.count++
    }
  }
}

// Vue3 Composition API
import { ref } from 'vue'

export default {
  setup() {
    const count = ref(0)
    const increment = () => count.value++
    
    return { count, increment }
  }
}
```

**优势**：
- 逻辑复用更方便
- 代码组织更清晰
- 更好的 TypeScript 支持

#### SQL.js（浏览器端数据库）

```javascript
import initSqlJs from 'sql.js'

// 初始化数据库（无需安装 MySQL）
const SQL = await initSqlJs()
const db = new SQL.Database()
```

**优势**：
- 开发时无需安装数据库
- 数据可直接持久化到文件
- 适合演示和小型项目

## 2.3 核心配置

### Vite 配置（vite.config.js）

```javascript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import path from 'path'

export default defineConfig({
  plugins: [vue()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src')
    }
  },
  server: {
    port: 5174,
    proxy: {
      '/api': {
        target: 'http://localhost:3001',
        changeOrigin: true
      }
    }
  }
})
```

### 路由配置（src/router/index.js）

```javascript
import { createRouter, createWebHistory } from 'vue-router'
import Dashboard from '../views/Dashboard.vue'
import Orders from '../views/Orders.vue'

const routes = [
  { path: '/', name: 'Dashboard', component: Dashboard },
  { path: '/orders', name: 'Orders', component: Orders }
]

export default createRouter({
  history: createWebHistory(),
  routes
})
```

### 状态管理（src/data/sharedData.js）

```javascript
import { ref } from 'vue'

// 订单数据（共享状态）
```

## 2.4 后端服务搭建

### 服务入口（server/index.js）

```javascript
const express = require('express')
const cors = require('cors')
const path = require('path')

const app = express()

// 中间件
app.use(cors())
app.use(express.json())
app.use(express.static(path.join(__dirname, '../dist')))

// API 路由
app.get('/api/orders', (req, res) => {
  // 返回订单数据
  res.json({ success: true, data: [] })
})

app.post('/api/orders', (req, res) => {
  // 创建订单
  res.json({ success: true })
})

// 启动服务
const PORT = 3001
app.listen(PORT, () => {
  console.log(`🚀 服务器已启动: http://localhost:${PORT}`)
})
```

## 2.5 前后端联调

### 开发模式

```
浏览器 (localhost:5174) 
    |
    | /api/xxx → 代理 → localhost:3001
    |
后端 API (localhost:3001)
```

### 请求示例

```javascript
// src/api/index.js
import axios from 'axios'

const api = axios.create({
  baseURL: '/api'
})

export const getOrders = () => api.get('/orders')
export const createOrder = (data) => api.post('/orders', data)
export const updateOrder = (id, data) => api.put(`/orders/${id}`, data)
export const deleteOrder = (id) => api.delete(`/orders/${id}')
```

### 组件中使用

```vue
<script setup>
import { onMounted, ref } from 'vue'
import { getOrders } from '@/api'

const orders = ref([])

onMounted(async () => {
  const res = await getOrders()
  orders.value = res.data
})
</script>

<template>
  <div v-for="order in orders" :key="order.id">
    {{ order.id }}
  </div>
</template>
```

## 2.7 本章小结

### 知识点回顾

1. **技术选型** - 根据项目需求、团队熟悉度、社区生态选择
2. **Vue3 + Vite** - 快速启动、热更新
3. **Express** - 轻量级后端框架
4. **前后端分离** - 代理解决跨域

### 常见问题

| 问题 | 解决 |
|-----|------|
| 端口被占用 | 修改 vite.config.js 中的 port |
| 跨域 | 使用 cors 中间件 |
| 代理不生效 | 检查 vite.config.js proxy 配置 |
