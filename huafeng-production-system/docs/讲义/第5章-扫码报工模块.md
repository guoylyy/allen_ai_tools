## 5.3 状态同步机制（核心重点）

### 问题分析

**问题**：报工完成后，排单页面如何实时更新状态？

**解决方案**：
1. 报工模块和排单模块共享同一份数据（sharedData）
2. 报工完成后调用同步函数
3. 触发响应式更新

### 同步函数实现

```javascript
/**
 * 同步工艺状态到订单
 * @param {boolean} forceCompleted - 是否强制设置为已完成
 */
const syncProcessStatusToOrder = (forceCompleted = false) => {
  if (!selectedOrder.value || !selectedProcess.value) return
  
  const items = currentItems.value
  
  // 统计完成数量
  const completedItems = items.filter(item => {
    const reported = item.reportedQty ? item.reportedQty[selectedProcess.value] : 0
    return reported >= item.quantity
  })
  
  const allCompleted = completedItems.length === items.length && items.length > 0
  const hasAnyReport = items.some(item => {
    const reported = item.reportedQty ? item.reportedQty[selectedProcess.value] : 0
    return reported > 0
  })
  
  // 确定状态
  let status = '待开始'
  if (forceCompleted || allCompleted) {
    status = '已完成'
  } else if (hasAnyReport) {
    status = '进行中'
  }
  
  // 更新到 order.processes
  selectedOrder.value.processes[selectedProcess.value] = {
    status: status,
    completedCount: completedItems.length,
    totalCount: items.length,
    canSchedule: status !== '已完成',
    items: items,
    startTime: processTimes.value[timeKey]?.startTime || null,
    endTime: processTimes.value[timeKey]?.endTime || null
  }
  
  // 同步到每个产品的 completedProcs
  items.forEach(item => {
    if (!item.completedProcs) item.completedProcs = {}
    const reported = item.reportedQty ? item.reportedQty[selectedProcess.value] : 0
    
    if (forceCompleted || (reported >= item.quantity && allCompleted)) {
      item.completedProcs[selectedProcess.value] = '已完成'
    } else if (reported > 0) {
      item.completedProcs[selectedProcess.value] = '进行中'
    }
  })
  
  // 重新计算下游工艺的阻塞状态
  computeProcessStatus(selectedOrder.value)
}
```

### 调用时机

```javascript
const finishWork = () => {
  // 检查是否全部报完
  const allCompleted = currentItems.value.every(item => 
    getReportedQty(item) >= item.quantity
  )
  
  if (!allCompleted) {
    alert('请先完成所有产品的报工！')
    return
  }
  
  // 标记所有产品为完成
  currentItems.value.forEach(item => {
    if (!item.completedProcs) item.completedProcs = {}
    item.completedProcs[selectedProcess.value] = '已完成'
  })
  
  // 强制设置为已完成
  syncProcessStatusToOrder(true)
  
  // 重新计算 canSchedule
  computeProcessStatus(selectedOrder.value)
  
  // 保存时间
  const timeKey = getTimeKey()
  const savedTimes = timeKey ? processTimes.value[timeKey] : null
  if (savedTimes && selectedOrder.value.processes[selectedProcess.value]) {
    selectedOrder.value.processes[selectedProcess.value].startTime = savedTimes.startTime
    selectedOrder.value.processes[selectedProcess.value].endTime = savedTimes.endTime
  }
}
```

## 5.4 时间追踪功能

### 问题分析

**问题**：用户输入报工数量后，开始时间和结束时间丢失了。

**原因**：每次同步状态都会重新创建对象，覆盖之前的时间。

**解决方案**：使用全局变量保存时间。

### 实现代码

```javascript
// 全局变量保存时间
const processTimes = ref({}) // { "orderId-processName": { startTime, endTime } }

// 获取时间key
const getTimeKey = () => {
  if (!selectedOrder.value || !selectedProcess.value) return null
  return `${selectedOrder.value.id}-${selectedProcess.value}`
}

// 开始报工时记录开始时间
const startWork = () => {
  // 初始化...
  
  // 记录开始时间
  const timeKey = getTimeKey()
  if (timeKey) {
    if (!processTimes.value[timeKey]) processTimes.value[timeKey] = {}
    processTimes.value[timeKey].startTime = new Date().toISOString()
  }
  
  // 同步状态
  selectedOrder.value.processes[selectedProcess.value] = {
    status: '进行中',
    startTime: processTimes.value[timeKey]?.startTime,
    // ...
  }
}

// 完成报工时记录结束时间
const finishWork = () => {
  // 同步状态（结束时间在 syncProcessStatusToOrder 中设置）
  syncProcessStatusToOrder(true)
  
  // 恢复时间（因为 computeProcessStatus 会覆盖）
  const timeKey = getTimeKey()
  const savedTimes = timeKey ? processTimes.value[timeKey] : null
  if (savedTimes && selectedOrder.value.processes[selectedProcess.value]) {
    selectedOrder.value.processes[selectedProcess.value].startTime = savedTimes.startTime
    selectedOrder.value.processes[selectedProcess.value].endTime = savedTimes.endTime
  }
}
```

### 计算用时

```javascript
const getDuration = (startTime, endTime) => {
  if (!startTime || !endTime) return null
  
  const start = new Date(startTime)
  const end = new Date(endTime)
  const diffMs = end - start
  
  const minutes = Math.floor(diffMs / 60000)
  const hours = Math.floor(minutes / 60)
  
  if (hours > 0) {
    return `${hours}小时${minutes % 60}分钟`
  }
  return `${minutes}分钟`
}
```
