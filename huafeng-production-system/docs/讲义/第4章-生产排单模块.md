## 4.1 排单模块概述

### 功能需求

生产排单模块是生产管理系统的核心，负责：
1. 展示所有订单及其生产状态
2. 根据工艺依赖关系判断是否可以排单
3. 管理每个工艺的开始/完成状态

### 核心概念

| 概念 | 说明 |
|-----|------|
| 可排单 | 前置工艺都已完成，可以开始生产 |
| 已阻塞 | 前置工艺未完成，无法开始 |
| 进行中 | 正在生产 |
| 已完成 | 全部完成 |

## 4.3 阻塞算法（核心重点）

### 问题分析

**问题**：如何判断某个工艺是否可以排单？

**业务规则**：
- 剪板是最开始工序，无前置依赖，可以排单
- 数冲需要等剪板完成后才能开始
- 折弯需要等剪板、数冲、激光下料都完成后才能开始

**数据模型**：
```javascript
const processDependencies = {
  '剪板': [],
  '数冲': ['剪板'],
  '激光下料': [],
  '折弯': ['剪板', '数冲', '激光下料'],
  '角钢线': ['剪板'],
  '切管机': [],
  '焊接': ['折弯', '角钢线', '切管机'],
  '喷涂': ['焊接'],
  '组装': ['喷涂']
}
```

### 算法实现

```javascript
/**
 * 计算工艺状态和是否可以排单
 * @param {Object} order - 订单对象
 */
const computeProcessStatus = (order) => {
  const processes = ['剪板', '数冲', '激光下料', '折弯', 
                     '角钢线', '切管机', '焊接', '喷涂', '组装']
  
  // 初始化 processes 对象
  if (!order.processes) {
    order.processes = {}
    processes.forEach(p => {
      order.processes[p] = {
        status: '待开始',
        canSchedule: false,
        completedCount: 0,
        totalCount: 0
      }
    })
  }
  
  // 遍历每个工艺
  processes.forEach((proc, index) => {
    const procData = order.processes[proc]
    
    // 如果已完成，跳过
    if (procData.status === '已完成') {
      return
    }
    
    // 获取前置依赖
    const prevProcs = processDependencies[proc] || []
    
    // 检查前置工艺是否都已完成
    const allPrevCompleted = prevProcs.every(prev => 
      order.processes[prev]?.status === '已完成'
    )
    
    // 判断是否可以排单
    if (allPrevCompleted) {
      procData.canSchedule = true
      procData.status = procData.status === '待开始' ? '可排单' : procData.status
    } else {
      procData.canSchedule = false
      procData.status = '已阻塞'
    }
  })
}
```

### 关键点解释

1. **获取前置依赖**：`processDependencies[proc]`
2. **检查是否完成**：遍历前置工艺列表，检查每个的状态
3. **全部完成才可排单**：使用 `every()` 方法

### 完整代码

```javascript
// src/data/sharedData.js

// 工艺依赖关系
export const processDependencies = {
  '剪板': [],
  '数冲': ['剪板'],
  '激光下料': [],
  '折弯': ['剪板', '数冲', '激光下料'],
  '角钢线': ['剪板'],
  '切管机': [],
  '焊接': ['折弯', '角钢线', '切管机'],
  '喷涂': ['焊接'],
  '组装': ['喷涂']
}

// 完整工艺列表
export const allProcesses = [
  '剪板', '数冲', '激光下料', '折弯', 
  '角钢线', '切管机', '焊接', '喷涂', '组装'
]

/**
 * 计算订单的工艺状态
 */
export const computeProcessStatus = (order) => {
  if (!order.processes) {
    order.processes = {}
  }
  
  allProcesses.forEach(proc => {
    if (!order.processes[proc]) {
      order.processes[proc] = {
        status: '待开始',
        canSchedule: false,
        completedCount: 0,
        totalCount: 0
      }
    }
    
    const procData = order.processes[proc]
    
    // 已完成的不用计算
    if (procData.status === '已完成') {
      return
    }
    
    // 获取前置依赖
    const prevProcs = processDependencies[proc] || []
    
    // 检查前置是否都完成
    const allPrevCompleted = prevProcs.every(prev => 
      order.processes[prev]?.status === '已完成'
    )
    
    // 设置状态
    if (allPrevCompleted) {
      procData.canSchedule = true
      if (procData.status === '待开始') {
        procData.status = '可排单'
      }
    } else {
      procData.canSchedule = false
      procData.status = '已阻塞'
    }
  })
}
```

## 4.4 排单交互实现

### 开始排单

```javascript
const confirmSchedule = (procName) => {
  const order = selectedOrder.value
  
  // 更新状态为进行中
  order.processes[procName].status = '进行中'
  order.processes[procName].startTime = new Date().toISOString()
  
  // 记录开始时间
  const now = new Date().toISOString()
  order.processes[procName].startTime = now
  
  // 重新计算下游工艺状态
  computeProcessStatus(order)
  
  // 保存到数据库
  saveOrder(order)
}
```

### 完成排单

```javascript
const completeProcess = (procName) => {
  const order = selectedOrder.value
  
  // 更新为已完成
  order.processes[procName].status = '已完成'
  order.processes[procName].endTime = new Date().toISOString()
  
  // 重新计算下游工艺状态
  computeProcessStatus(order)
  
  // 保存
  saveOrder(order)
}
```

## 4.5 进度计算

### 整体进度百分比

```javascript
const getProgress = (order) => {
  const procs = ['剪板', '数冲', '激光下料', '折弯', 
                 '角钢线', '切管机', '焊接', '喷涂', '组装']
  
  let completed = 0
  procs.forEach(p => {
    if (order.processes?.[p]?.status === '已完成') {
      completed++
    }
  })
  
  return Math.round((completed / procs.length) * 100)
}
```
