# 客户自助进度查询系统 - 需求文档

## 一、需求概述

### 目标
客户通过微信服务号H5页面自助查看订单生产进度，提升信息透明度，减少客户催单咨询。

### 用户场景
- 客户想知道"我的电梯做得怎么样了？"
- 客户想知道"能否按时发货？"
- 客户想看"每个工艺/零件的生产情况"

---

## 二、功能需求

### 2.1 登录方式
- [ ] 微信服务号内H5页面
- [ ] 输入合同号/订单号绑定
- [ ] 绑定后微信一键登录

### 2.2 合同列表
- [ ] 客户可见自己名下所有合同
- [ ] 显示：合同号、项目名、交货期、状态、进度

### 2.3 进度详情
- [ ] 整体进度百分比 + 进度条
- [ ] 各工艺生产状态（待开始/进行中/已完成）
- [ ] 每个零件的生产情况

### 2.4 详情层级

```
合同详情
├── 整体进度 (75%)
├── 工艺进度
│   ├── 剪板 ✓ 已完成
│   ├── 数冲 ✓ 已完成  
│   ├── 折弯 🔄 进行中 (3/10)
│   ├── 焊接 ⏳ 待开始
│   └── ...
└── 零件详情
    ├── 零件A ✓ 已完成
    ├── 零件B ✓ 已完成
    ├── 零件C 🔄 进行中
    └── ...
```

---

## 三、非功能需求

### 3.1 界面风格
- 谷歌Material Design风格
- 简约、干净
- 移动端优先（H5）

### 3.2 微信集成
- 微信服务号菜单入口
- 微信授权登录
- 消息推送（可选）

### 3.3 性能要求
- 页面加载 < 2秒
- 实时数据展示

---

## 四、数据来源

数据来自生产管理系统（现有系统）

| 数据表 | 用途 |
|-------|------|
| orders | 订单基本信息 |
| products | 产品/零件信息 |
| production_records | 生产记录 |
| processes | 工艺状态 |

## 六、技术方案

### 6.1 架构
```
┌─────────────┐     ┌─────────────┐
│ 微信服务号  │────▶│  H5前端   │
└─────────────┘     └──────┬──────┘
                          │
                          ▼
                 ┌─────────────┐
                 │ 生产管理系统 │
                 │   API      │
                 └─────────────┘
```

### 6.2 技术栈
- 前端：Vue3 + Vant UI（移动端组件库）
- 后端：复用现有Express API
- 微信：服务号网页授权
