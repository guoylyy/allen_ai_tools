# Daily Project Summary

## 指令

### 第一步：获取项目信息
1. 首先确认项目路径（如果用户没提供）
2. 确认报告的时间范围（默认昨天至今）

### 第二步：收集修改信息
1. 获取 Git 提交记录：
   ```bash
   git log --since="昨天 00:00" --until="今天 00:00" --pretty=format:"%h | %an | %ad | %s" --date=short
   ```
2. 获取新增/删除/修改的文件：
   ```bash
   git diff --stat $(git yesterday) HEAD
   ```
3. 如果有未提交的更改：
   ```bash
   git status --short
   ```

### 第三步：生成报告
根据不同受众调整：

**技术团队版**：
- 提交数量、提交者
- 每个提交的内容和变更文件
- 代码行数变化（+/-）
- 遇到的问题和解决方案

**管理版**：
- 完成了哪些功能/任务
- 进展是否符合计划
- 遇到的阻塞问题
- 下一步计划

### 第四步：输出格式
```markdown
# 📊 [项目名称] 日报 - [日期]

## 📈 概览
- 提交数：X
- 变更文件：X
- 代码变化：+X / -X

## ✅ 完成事项
- [任务1] - 状态
- [任务2] - 状态

## 🚧 进行中
- [任务]

## ⚠️ 问题/阻塞
- 问题描述

## 📝 提交详情
| 提交 | 作者 | 描述 |
|------|------|------|
| xxx  | xxx  | xxx  |

## 📅 明日计划
- [计划1]
- [计划2]
```

## 示例

### 示例 1：团队内部日报
用户说："生成今天的项目日报"
操作：
1. 执行 git log 获取今日提交
2. 执行 git diff 获取变更统计
3. 生成包含技术细节的报告

### 示例 2：给老板的进度汇报
用户说："给我一份今天的进度汇报"
操作：
1. 执行 git log 获取今日提交
2. 转换为业务语言描述
3. 生成简洁的管理版报告

## 排错

### 问题：找不到 Git 仓库
原因：当前目录不是 Git 仓库
解决方案：请提供正确的项目路径

### 问题：没有今日提交
原因：当天没有提交
解决方案：提示用户并询问是否查看昨日或更早的记录
